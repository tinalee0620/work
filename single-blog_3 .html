/ 作品介紹

KIDOWE是一個能讓大家能夠更了解行程，共同討論 並及時提出想法和疑慮，並且在旅遊時能順利的解決 各種路線或交通問題的app，透過共同編輯，投票和 聊天室等功能，讓大家能即時討論且做出決定。行程的地圖化也讓路線一目了然，增加規劃的簡易性，讓 大家能輕鬆規劃，歡樂出遊。此外為了增加趣味性， 旅程的每一天都會打造一棟建築物，讓旅行成就實體化， 並出現在行程頁面上增加趣味性和成就感。

/ 設計理念

- 趣味呈現 : 跳脫多數安排行程 App 的排版方式將介面以有趣的方式呈現

- 簡單操作 : 以最直覺、簡易、流暢的方式安排行程。

- 輕鬆規劃 : 透過共同編輯、投票、聊天室等方式了解大家的需求與想法, 完成最有效率、輕鬆的規劃。

/ 構想與特色

- 投票、轉盤 : 在旅行時解決選擇障礙和意見不同的狀況

- 聊天室 : 不用換 App 也可以隨時與旅伴聯繫

- 問路卡 : 語言不通時,不需查詢翻譯、比手畫腳，就能清楚讓對方知道你 想去的地方

- 旅行清單 : 內建旅行必備用品的清單，整理行李不再手忙腳亂、忘東忘 西，輕鬆踏上旅程

/ 設計產出

主要功能

此四頁為主頁，地圖，聊天和記帳頁面行程除了會在主頁上顯示外，在地圖上也會以座標顯示並連線，讓使用者可以知道今天行程的路線。聊天頁面則是可以讓使用者不用切換到其他聊天程式中，可以即時了聯絡處理問題。記帳工能可以清楚的統計這趟行程的所有花費，可以依照天數或是分類去計算。

建立旅程

我們盡量的簡化建立旅程的步驟，讓使用者不會因為要填寫太多資訊而不想使用。再邀請朋友加入時，除了可以邀請有下載這個app的人除外，還可以用email去邀請沒有這個app的人來使用。

建立玩旅程後，便會形成一塊地層，點進地層，就會來到主頁。

加入行程

按下新增鍵後，就可以依照行程種類去選擇，每一個景點或是美食都會有更詳細的資訊，讓使用者不用再另外去查詢，最後，自訂的選項是可以手動輸入，或是不知道如何歸類的行程，可以自由地去運用。

票選活動

再決定行程時，常常會有意見不合的時候，所以有了投票的選項，將景點往左滑，可以選擇加入行程或是加入投票，如果是投票的話，會在主頁上顯示投票的一個行程區塊，讓大家去投票。

/ 自我審視與改善

* 第一次嘗試用Sketch去設計一個app，雖然基本的功能都知道，但對於symbol和override的使用很陌生，導致設計檔凌亂，要改配色也要逐一改。

解決方法：我在之後的公司實習中，熟悉了symbol的功能，和如何整理設計檔，讓工程師能輕易看懂。

* 沒有仔細的考慮程式是否能完成我們想要的功能，造成後期在寫程式時，要把一些功能刪除，同時也要修改頁面。

解決方法：在設計每項功能及操作流程都要更深思熟慮，也要再畫wireframe時跟負責程式的同學討論，確認功能的擺放位置和功能開發的可能性，再進一步去畫設計稿。
